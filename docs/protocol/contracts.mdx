Axios’s execution layer is a set of **Solidity** contracts on **Somnia** (chain ID **50312**). Addresses **change every time you redeploy** — copy them from the `forge script` output into `.env.local` (see [Contract deployment](/docs/operations/contract-deployment)).

## Deployed stack

| Contract | Role |
|----------|------|
| **ComplianceRegistry** | Owner-managed allow/block compliance flags used during registration and checks. |
| **PayrollTreasury** | Holds sUSDC per employer; **deposit**, **withdraw** (available only), gas budget; batcher locks/releases. |
| **PayrollBatcher** | Executes batch payroll; authorized agents; interacts with treasury + token transfers. |
| **EmployeeRegistry** | Maps employees to employers and compliance state. |
| **StreamVesting** | Salary streaming / vesting claims (sUSDC). |
| **YieldRouter** | Treasury yield routing (where configured). |

## Environment variables

The Next.js app reads deployment addresses from:

- `NEXT_PUBLIC_COMPLIANCE_REGISTRY`
- `NEXT_PUBLIC_PAYROLL_TREASURY`
- `NEXT_PUBLIC_PAYROLL_BATCHER`
- `NEXT_PUBLIC_EMPLOYEE_REGISTRY`
- `NEXT_PUBLIC_STREAM_VESTING`
- `NEXT_PUBLIC_YIELD_ROUTER`

**sUSDC** token (canonical on testnet): `NEXT_PUBLIC_SUSDC_ADDRESS` — see Somnia docs for the current reference address.

Do **not** hardcode old Moderato/Tempo addresses; always align with your `.env.local` and [Somnia Explorer](https://shannon-explorer.somnia.network).
