The Axios app is a **Next.js 15** App Router project. It talks to **Supabase** (Postgres + RLS), **Mezo** via RPC, and issues **session JWTs** after **SIWE** (Sign-In With Ethereum) verification.

## Prerequisites

1. **Supabase** — Create a project. Apply the schema from `scripts/schema.sql` (tables: `employers`, `employees`, `payroll_runs`, `payment_items`, etc.). Use the **service role** key only on the server.
2. **Contract addresses** — Deploy contracts to Mezo testnet and set all `NEXT_PUBLIC_*` addresses in `.env.local`.
3. **Auth** — No Privy app ID is required for the current wallet flow. The app uses **RainbowKit + wagmi + viem** on Mezo testnet; login uses **SIWE** and an **httpOnly** `axios-token` cookie. Set `SESSION_JWT_SECRET` (long random string).

## Hosting (e.g. Vercel)

Set environment variables in the hosting dashboard to match `.env.local` (production values). Build:

```bash
pnpm install
pnpm build
```

## Verification

- Employer dashboard loads treasury and **deposit / withdraw** against the configured `NEXT_PUBLIC_PAYROLL_TREASURY`.
- Employee portal: **employee** wallet session can call `/api/me/employee` and `/api/me/payments`; **employer-only** sessions receive **403** on those routes.

## MPP

`/api/mpp/*` routes are available for **HTTP 402** / x402 integrations; wire payment verification in `lib/mpp.ts` for strict production gating.
